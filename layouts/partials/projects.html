DoodleScript (2016)

How would computer programming look like if we could reinvent it now from the ground up?

With this project I wanted to radically question the way we program. Why do we type code at all? Maybe we could just use visual symbols. And then maybe we could even write programs by just doodling manual sketches that the computer would interpret and run for us —after all, computers should be getting smart enough to be able to interpret human language instead of the other way around.

Together with my colleague Benjamin Skirlo we won the main prize of the Angelhack Berlin 2016 hackathon with a proof of concept of this idea.

Garbuix (2014)

Garbuix is my Processing-based tool to explore visually interesting behaviours of 2D systems.

The user can toggle and influence a set of rather simple behavioral and rendering rules applied to a particle system. (For example, attraction/ repulsion, triangulation...)

It is interesting to see how a small set of rules can produce dramatically different effects on just a set of nodes randomly distributed on a 2D space.

https://github.com/jperals/garbuix

Big Bother (2014)

Big Bother (sic) is a generative sketch that uses computer vision. It features a giant mutant eye that has the ability to grow a new pupil for each moving target it detects (i.e, each person in front of it) and track it individually.

This piece was played using a huge screen in the center of Madrid as part of MediaLab Prado’s #ProgramaLaPlaza contest in 2014.

https://player.vimeo.com/video/100365334#t=31m09s

Terra Cognita (2012-2013)

As I moved to Berlin in 2012, I started tracking my movements and then wrote an OpenFrameworks app that let me visualize my process of getting to know a new place.

My walkabouts were rendered at high speed mapping the speed to colours, so I could even tell where and when I had been commuting with the train (red), walking (green), taking a tram (purple) or even riding a bike (blue).

https://github.com/jperals/terra-cognita
